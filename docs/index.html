Dependencias y prompts, auditados en local

zscan consulta OSV (npm, PyPI, Go, RubyGems, Maven), genera zscan - Reporte de vulnerabilidades (HTML, Markdown, JSON), evalúa prompts[] con zscan.yaml y opcionalmente un LLM. No modifica tu repositorio.

Inicio rápido

Cinco pasos desde cero hasta abrir el informe. El proyecto que escaneás puede ser en cualquier lenguaje soportado por los lockfiles; solo hace falta Node.js 18+ para ejecutar zscan.

  1. Requisitos

    Node.js 18 o superior. Comprobá con node -v.

  2. Obtener zscan

    Cloná el repositorio (o instalalo como devDependency desde GitHub; ver README). 

    git clone https://github.com/angievic/zscan.git
cd zscan
  3. Instalar dependencias y compilar 
    npm install
npm run build

    En la raíz de zscan podés usar ./zscan (instala y compila si hace falta).

  4. Configurar y escanear tu app

    Desde la raíz del proyecto a auditar (no hace falta que sea un repo Node): 

    node /ruta/a/zscan/dist/cli.js init --root .
node /ruta/a/zscan/dist/cli.js scan-all --root .

    init crea o actualiza zscan.yaml. scan-all deja un paquete en zscan-runs/<id>-scan/ (ignorá esa carpeta en git si no querés versionar informes).

  5. Abrir el informe

    En la carpeta del run, abrí report.html en el navegador. Ahí verás resumen de vulnerabilidades (CVSS), runtimes, secretos/auth heurísticos, prompts y más. También tenés informe.md y report.json.

Salida CI
El comando termina con código 1 si hay hallazgos que marcan riesgo (OSV, prompts, etc.).

Características

Multi-ecosistema + OSV

Un solo comando puede combinar lockfiles presentes y consultar vulnerabilidades conocidas.

Informe unificado

HTML por pestañas, Markdown y JSON: dependencias, CVSS, enlaces de contexto web, escaneo de secretos heurístico y prompt-scan.

Prompts e init

zscan init propone globs según docs, agentes y código con literales tipo LLM. Reglas YAML + heurísticas + LLM si llm.enabled.

Offline OSV

scan --offline o ZSCAN_OFFLINE=1 para usar solo caché local.

Referencias CVE

Por defecto se intentan extractos desde URLs de advisories (caché en disco). Omite con --no-enrich-docs.

LLM opcional

Ollama, OpenAI, Gemini (compatible), Claude. config, llm-probe, serve para POST /scan.

Instalación desde el código

Mismo flujo que el paso 2–3 del inicio rápido. No hace falta publicar en npm para usar la herramienta. 

git clone https://github.com/angievic/zscan.git
cd zscan
npm install
npm run build

Opcional: chmod +x zscan y ./zscan scan-all --root /tu/proyecto. Variable ZSCAN_ROOT si usás el script sin --root.

Comandos útiles

node dist/cli.js init --root .
node dist/cli.js scan-all --root .
node dist/cli.js scan-all --root . --no-enrich-docs
node dist/cli.js scan-all --root . --no-prompt-llm
node dist/cli.js config --root .
node dist/cli.js llm-probe --root .
node dist/cli.js prompt-scan --root .
node dist/cli.js serve --port 8787

./zscan scan-all

npm test

Referencia completa: README → Referencia de configuración.

Referencias CVE en el informe

OSV aporta metadatos y enlaces; el texto del advisory aporta contexto. En cada scan (salvo --no-enrich-docs) se descargan esas URLs, se reduce HTML a texto y se añaden extractos al informe, con caché en disco. 

node dist/cli.js scan-all --root .
node dist/cli.js scan-all --root . --no-enrich-docs

Variables ZSCAN_ENRICH_CACHE_DIR, caché OSV con ZSCAN_OSV_CACHE_DIR. API POST /scan: "enrichDocs": false omite scraping. Más contexto en vision-agente-seguridad.md.
